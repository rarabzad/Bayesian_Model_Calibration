Bayesian Calibration for Hydrologists

Part 1 Introduction

What Are Hydrologic Models?

Fundamental tools for simulating:

  • Streamflow generation and flood routing
  • Snow accumulation and melt dynamics
  • Groundwater exchange and soil moisture
  • Evapotranspiration processes

Applications

Flood forecasting Drought assessment Reservoir management Climate impact studies Water resources planning

Models are uncertain representations of reality calibration is essential for reliable predictions.

Sources of Uncertainty

Parameter Uncertainty

Cannot measure soil hydraulic conductivity, recession constants, or snowmelt factors at watershed scale.

Forcing Uncertainty

Precipitation & temperature inputs contain measurement errors and spatial interpolation errors.

Structural Uncertainty

Model equations simplify real-world processes lumped models aggregate spatial variability.

Observation Uncertainty

Streamflow measurements are subject to rating curve errors, sensor inaccuracies, sampling limits.

Traditional Calibration: Limitations

What it does

Minimize a performance metric (NSE, KGE, RMSE) single best parameter set.

Key problems

Problem Description
Equifinality Many parameter sets produce equally good fits
No uncertainty Single estimate ignores parameter uncertainty
No prior knowledge Cannot formally incorporate expert knowledge
Point predictions No probabilistic forecast envelopes

Bayesian calibration solves all of these by treating parameters as probability distributions.

Why Bayesian Calibration?

Advantages over traditional optimization

  1. Full uncertainty quantification posterior distributions, not point estimates
  2. Formal prior knowledge field measurements, literature, expert judgment
  3. Uncertainty propagation through model to predictions
  4. Probabilistic predictions credible intervals with coverage guarantees
  5. Iterative learning update beliefs as new data arrives

Traditional calibration finds a single best parameter set. Bayesian calibration finds the full distribution of plausible parameter sets.

The Bayesian Workflow

Prior knowledge  p()         physical bounds, literature
       +
Observed data    Q_obs        streamflow gauge record
       
Bayes' Theorem:  p(|D)  p(D|)  p()
       
Posterior        p(|D)       updated parameter distributions
       
Probabilistic predictions     credible intervals, ensemble runs

Key insight: Parameters with high prior plausibility AND high likelihood receive the highest posterior probability.

Part 2 Bayesian Theory

Bayes Theorem

\[p(\theta \mid D) = \frac{p(D \mid \theta) \, p(\theta)}{p(D)}\]

Symbol Name Meaning
\(p(\theta \mid D)\) Posterior Updated knowledge after data
\(p(D \mid \theta)\) Likelihood How well parameters explain data
\(p(\theta)\) Prior Initial knowledge before data
\(p(D)\) Evidence Normalizing constant

In practice: \(p(\theta \mid D) \propto p(D \mid \theta) \cdot p(\theta)\)

The normalizing constant \(p(D)\) is not needed for MCMC sampling it cancels in acceptance ratios.

The Bayesian Update Process



1. Define prior  p()           physical knowledge, literature
        
2. Run model     Q()         GR4J or other simulator
        
3. Evaluate likelihood p(D|)   how well does  explain observations?
        
4. Compute posterior p(|D)  p(D|)  p()
        
5. Sample posterior via MCMC
        
6. Posterior predictive checks & residual diagnostics

Key insight: Parameters with both high prior plausibility AND high likelihood receive highest posterior probability.

Bayesian Inference in Hydrology

Model structure

\[\hat{Q}_t = f(\theta,\, X_t,\, S_0)\]

  • \(\theta\) parameters (soil, snow, groundwater)
  • \(X_t\) forcings (precipitation, temperature)
  • \(S_0\) initial states

Residuals

\[\varepsilon_t = Q_t^{\text{obs}} - \hat{Q}_t\]

The likelihood is built from assumptions about the distribution of \(\varepsilon_t\).

Bayesian Inference in Hydrology

Prior choices

Parameter type Suggested prior
Strictly positive (X1, X3) Log-normal or Uniform on log-scale
Bounded [0,1] recession Beta or truncated Normal
Possibly negative (X2) Normal or Uniform

Part 3 Likelihood Functions

Homoscedastic Gaussian Likelihood

Assumption

\[\varepsilon_t \sim \mathcal{N}(0,\,\sigma^2) \quad \text{(constant variance, independent)}\]

Log-likelihood

\[\log p(D \mid \theta, \sigma) = -\frac{T}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_{t=1}^{T}(Q_t^{\text{obs}} - \hat{Q}_t)^2\]

Interpretation

Simple, easy to implement
Good starting point

Equal weight to all flows
Ignores temporal dependence
Underestimates uncertainty

Heteroscedastic + AR(1) Likelihood

Flow-dependent variance

\[\sigma_{\varepsilon(t)} = a \cdot \hat{Q}_t + b\]

Standardized residuals

\[\eta_t = \frac{\varepsilon_t}{\sigma_{\varepsilon(t)}} = \frac{Q_t^{\text{obs}} - \hat{Q}_t}{\sigma_{\varepsilon(t)}}\]

AR(1) process on \(\eta_t\)

\[\eta_1 \sim \mathcal{N}\!\left(0,\, \tfrac{1}{1-\phi^2}\right), \qquad \eta_t \mid \eta_{t-1} \sim \mathcal{N}(\phi\,\eta_{t-1},\, 1), \quad t \geq 2\]

Heteroscedastic Log-Likelihood

\[\begin{aligned} \log p(D \mid \theta, a, b, \phi) = & -\tfrac{1}{2}\log\!\left(\tfrac{2\pi}{1-\phi^2}\right) - \tfrac{\eta_1^2(1-\phi^2)}{2} \\ &- \tfrac{T-1}{2}\log(2\pi) - \tfrac{1}{2}\sum_{t=2}^{T}(\eta_t - \phi\,\eta_{t-1})^2 \\ &\underbrace{- \sum_{t=1}^{T}\log(\sigma_{\varepsilon(t)})}_{\text{Jacobian adjustment}} \end{aligned}\]

The Jacobian term \(-\sum\log\sigma_{\varepsilon(t)}\) is mandatory. Omitting it biases all parameter estimates and invalidates uncertainty quantification.

Why the Jacobian Matters

When transforming \(\varepsilon_t \to \eta_t = \varepsilon_t / \sigma_{\varepsilon(t)}\), the change-of-variables rule requires: \[p(\boldsymbol{\varepsilon}) = p(\boldsymbol{\eta})\prod_{t=1}^{T}\sigma_{\varepsilon(t)}\]

In log space: \(\log p(\boldsymbol{\varepsilon}) = \log p(\boldsymbol{\eta}) - \sum_{t=1}^{T}\log\sigma_{\varepsilon(t)}\)

Consequences of omitting Jacobian

Effect Impact
Biased \(a\), \(b\) estimates Wrong uncertainty scaling
Incorrect model comparison Likelihoods on different scales
Prediction intervals off Under/over-coverage 
# Jacobian term in R  always include this line
sigma_t       <- pmax(1e-6, a * Q_sim + b)  # flow-dependent std devs
jacobian_term <- -sum(log(sigma_t))          # subtract from log-likelihood

Prior Distributions

\[\beta_0 \sim \mathcal{N}(0,10^2), \quad \beta_1 \sim \mathcal{N}(0,10^2), \quad \sigma \sim \text{HalfNormal}(5)\]

log_prior <- function(theta) {
  beta0 <- theta[1]; beta1 <- theta[2]; sigma <- theta[3]

  if (sigma <= 0) return(-Inf)   # hard positivity constraint

  log_p_beta0 <- dnorm(beta0, mean = 0, sd = 10, log = TRUE)
  log_p_beta1 <- dnorm(beta1, mean = 0, sd = 10, log = TRUE)
  # HalfNormal: N(0, 5) truncated to  > 0  correct with +log(2)
  log_p_sigma <- dnorm(sigma, mean = 0, sd = 5, log = TRUE) + log(2)

  return(log_p_beta0 + log_p_beta1 + log_p_sigma)
}

Why weakly informative priors?

  • Regularize: prevent absurd parameter values
  • Not so strong they overwhelm the data
  • HalfNormal enforces \(\sigma > 0\) naturally; +log(2) re-normalizes the truncated density

Log-Likelihood & Log-Posterior

# Log-likelihood (homoscedastic Gaussian)
log_likelihood <- function(theta) {
  beta0 <- theta[1]; beta1 <- theta[2]; sigma <- theta[3]
  y_pred <- beta0 + beta1 * x
  sum(dnorm(y, mean = y_pred, sd = sigma, log = TRUE))
}

# Log-posterior = log-prior + log-likelihood
log_posterior <- function(theta) {
  lp <- log_prior(theta)
  if (is.infinite(lp)) return(lp)   # skip likelihood if prior = 0
  lp + log_likelihood(theta)
}

Sanity Check Before MCMC

# Always verify the posterior evaluates correctly at a known point
theta_true <- c(beta0_true, beta1_true, sigma_true)

# Should be a finite negative number; -Inf means something is wrong

Always check if(is.infinite(lp)) return(lp) before evaluating the likelihood this prevents -Inf + Inf = NaN which silently breaks MCMC.

Part 5 GR4J Bayesian Calibration

GR4J Model Overview

GR4J Gnie Rural 4 paramtres Journalier a parsimonious daily rainfall-runoff model.

Precipitation (P) + PET + Temperature (T)
        
  [Snow Module]   TT, DDF parameters  rain/snow split + degree-day snowmelt
        
  [Production Store X1]  soil moisture accounting, partitions P into ET and runoff
        
  [Unit Hydrographs UH1 (90%) and UH2 (10%)]  controlled by X4 (flow timing)
        
  [Routing Store X3]  baseflow and groundwater recession
        
  [Groundwater Exchange X2]  net gain (positive) or loss (negative) to deep aquifer
        
  Simulated Streamflow Q  (m/s)

Model Parameters

Symbol Meaning Units Typical Range
X1 Production store capacity mm 1 3000
X2 Groundwater exchange coefficient 5 to 10
X3 Routing store capacity mm 1 2000
X4 Unit hydrograph time base days 0.1 60
TT Temperature threshold (rain/snow split) C 15 to 10
DDF Degree-day melt factor mm C day 0 20
Residual standard deviation m/s 0.001 200

is an error model parameter estimated jointly with the hydrological parameters a key Bayesian advantage over deterministic calibration.

Parameter Physical Interpretation

X1 (Production store): Low (< 100 mm) flashy, high runoff coefficients, shallow soils. High (> 500 mm) deep soils, sustained baseflows. Effectively absorbs some structural model error.

X2 (Groundwater exchange): Positive = net gain from neighbouring aquifers (karst, regional upwelling). Negative = losses to deep percolation or adjacent basins. Often poorly identifiable consider fixing to 0 if local hydrogeology indicates minimal exchange.

X3 (Routing store): Controls recession slope and low-flow support. Larger slower recessions. Informed by recession analysis of dry-spell flow data.

X4 (UH time base): Scales with catchment concentration time. Small, steep: < 2 days. Large, flat: > 10 days.

TT (Temperature threshold): Typically 2 to +2 C. Maritime climates closer to +2. Continental near 0 C or below.

DDF (Degree-day factor): Open sites: 36 mm C day. Forested: 13. Regional snow literature often provides suitable prior ranges.

# Log-prior: uniform within bounds
log_prior <- function(theta) {
  if (any(theta < lb) || any(theta > ub)) return(-Inf)
  sum(dunif(theta, lb, ub, log = TRUE))
}

Run prior predictive checks before calibrating: sample \(\theta\) from the prior, simulate hydrographs, and verify they are physically plausible. Revise bounds if simulated flows look unrealistic.

Homoscedastic Likelihood

source("https://raw.githubusercontent.com/rarabzad/Bayesian_Model_Calibration/refs/heads/main/docs/gr4j_sim.R")

log_likelihood_homo <- function(theta) {
  hydro_params <- theta[1:6]; sigma <- theta[7]
  if (sigma <= 0) return(-Inf)
  Q_sim_xts <- gr4j_sim(P_xts, T_xts, PET_xts, hydro_params, area_km2)
  merged <- merge(Q_sim_xts, Q_obs_xts, all = FALSE)
  merged <- merged[!apply(is.na(merged), 1, any), ]
  Q_sim  <- as.numeric(merged[, 1]); Q_obs <- as.numeric(merged[, 2])
  if (length(Q_obs) == 0) return(-Inf)
  sum(dnorm(Q_obs, mean = Q_sim, sd = sigma, log = TRUE))
}

Heteroscedastic + AR(1) Likelihood

log_likelihood_hetero_ar1 <- function(theta) {
  hydro_params <- theta[1:6]; a <- theta[7]; b <- theta[8]; phi <- theta[9]
  if (a < 0 || b <= 0 || abs(phi) >= 1) return(-Inf)
  Q_sim_xts <- gr4j_sim(P_xts, T_xts, PET_xts, hydro_params, area_km2)
  merged    <- merge(Q_sim_xts, Q_obs_xts, all = FALSE)
  merged    <- merged[!apply(is.na(merged), 1, any), ]
  Q_sim <- as.numeric(merged[, 1]); Q_obs <- as.numeric(merged[, 2])
  sigma_t <- pmax(1e-6, a * Q_sim + b)           # flow-dependent _(t)
  eta     <- (Q_obs - Q_sim) / sigma_t            # standardized residuals
  ll <- dnorm(eta[1], mean = 0, sd = 1 / sqrt(1 - phi^2), log = TRUE)
  for (t in 2:length(eta))
    ll <- ll + dnorm(eta[t], mean = phi * eta[t - 1], sd = 1, log = TRUE)
  ll - sum(log(sigma_t))                          # Jacobian (mandatory)
}

Part 6 Posterior Predictive Checks

Theory & Procedure

Posterior predictive distribution

\[p(Q_{\text{pred}} \mid Q_{\text{obs}}) = \int p(Q_{\text{pred}} \mid \theta)\, p(\theta \mid Q_{\text{obs}})\, d\theta\]

Includes two sources of uncertainty

Source Captured by
Parameter uncertainty Different \(\theta\) values across posterior samples
Residual variability Noise \(\varepsilon \sim p(\cdot \mid \theta)\) added to each simulation

Theory & Procedure

Procedure

  1. Sample \(\theta^{(i)}\) from MCMC chain

  2. Run GR4J \(Q_{\text{sim}}^{(i)}\)

  3. Draw \(Q_{\text{pred},t}^{(i)} = Q_{\text{sim},t}^{(i)} + \varepsilon_t^{(i)}\)

  4. Summarise ensemble; compare to \(Q_{\text{obs}}\)

Predictive Ensemble (Heteroscedastic AR(1))

for (i in seq_len(n_pred)) {
  theta_i <- samples[pred_idx[i], ]
  a <- theta_i[7]; b <- theta_i[8]; phi <- theta_i[9]
  Q_sim_xts <- gr4j_sim(P_xts, T_xts, PET_xts, theta_i[1:6], area_km2)
  merged    <- merge(Q_sim_xts, Q_obs_xts, all = FALSE)
  Q_sim     <- as.numeric(merged[, 1])
  sigma_t <- pmax(1e-6, a * Q_sim + b)    # flow-dependent std devs
  sigma_y <- sqrt(1 - phi^2)              # innovation std dev
  epsilon    <- numeric(length(Q_sim))
  epsilon[1] <- sigma_t[1] * rnorm(1)
  for (t in 2:length(Q_sim)) {
    eta_prev   <- epsilon[t - 1] / sigma_t[t - 1]
    eta_t      <- phi * eta_prev + rnorm(1, 0, sigma_y)
    epsilon[t] <- sigma_t[t] * eta_t
  }
  Q_pred_samples[i, ] <- Q_sim + epsilon
}

Why Diagnose Residuals?

Violations of likelihood assumptions cause:

Violation Consequence
Heteroscedasticity Biased , wrong prediction intervals
Autocorrelation Effective sample size overstated, biased parameters
Non-normality Credible intervals miscalibrated
Systematic bias Parameters compensate for missing processes

Workflow 1: Log-Likelihood & Log-Posterior

log_likelihood_homo <- function(theta) {
  hydro_params <- theta[1:6]; sigma <- theta[7]
  if (sigma <= 0) return(-Inf)
  Q_sim_xts <- gr4j_sim(P_xts, T_xts, PET_xts, hydro_params, area_km2)
  merged    <- merge(Q_sim_xts, Q_obs_xts, all = FALSE)
  merged    <- merged[!apply(is.na(merged), 1, any), ]
  Q_sim <- as.numeric(merged[, 1]); Q_obs <- as.numeric(merged[, 2])
  if (length(Q_obs) == 0) return(-Inf)
  sum(dnorm(Q_obs, mean = Q_sim, sd = sigma, log = TRUE))
}

log_posterior <- function(theta) {
  lp <- log_prior(theta)
  if (is.infinite(lp)) return(-Inf)
  lp + log_likelihood_homo(theta)
}

Workflow 2: Heteroscedastic + AR(1) Likelihood

log_likelihood_hetero_ar1 <- function(theta) {
  hydro_params <- theta[1:6]; a <- theta[7]; b <- theta[8]; phi <- theta[9]
  if (a < 0 || b <= 0 || abs(phi) >= 1) return(-Inf)
  Q_sim_xts <- gr4j_sim(P_xts, T_xts, PET_xts, hydro_params, area_km2)
  merged    <- merge(Q_sim_xts, Q_obs_xts, all = FALSE)
  merged    <- merged[!apply(is.na(merged), 1, any), ]
  Q_sim <- as.numeric(merged[, 1]); Q_obs <- as.numeric(merged[, 2])
  sigma_t <- pmax(1e-6, a * Q_sim + b)
  eta     <- (Q_obs - Q_sim) / sigma_t
  ll <- dnorm(eta[1], mean = 0, sd = 1 / sqrt(1 - phi^2), log = TRUE)
  for (t in 2:length(eta))
    ll <- ll + dnorm(eta[t], mean = phi * eta[t - 1], sd = 1, log = TRUE)
  ll - sum(log(sigma_t))   # Jacobian (mandatory)
}

Summary & Best Practices

Key Takeaways

Bayesian advantages

  1. Full parameter uncertainty
  2. Probabilistic predictions
  3. Prior knowledge encoded formally
  4. Equifinality represented
  5. Iterative learning as data grow

Key Takeaways

MCMC success criteria

  • Acceptance rate: 0.15 0.30
  • ESS > 1000 per parameter
  • Gelman-Rubin R < 1.1
  • Trace plots: stationary, no trends

Run all diagnostics before interpreting posteriors a passing acceptance rate alone is insufficient.

Key Takeaways

Likelihood selection

  1. Start: homoscedastic Gaussian
  2. Funnel in residuals heteroscedastic
  3. ACF significant add AR(1)
  4. Heavy tails Student-t
  5. Always include Jacobian when transforming

Prior guidance

  1. Weakly informative: prevents absurdity, doesnt dominate
  2. Bounds from physical knowledge
  3. Run prior predictive checks first
